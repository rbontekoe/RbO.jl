2 - Building applications with Julia

Building applications with Julia

Courseware Description

Target audience

The target audience for the course Building applications with Julia are application developers who will be using Julia in a structured way.

Summary description

Students will learn the technique to create a module that can to be used in container-based applications. The workshop starts by creating the domain layer. Besides, the domain objects are the functions defined for the API layer and the Infrastructure layer. Students learn to create documentation for the application that also tests the code examples.

Courseware goals

By the end of the course, you should be able to:

  • Create a model with Domain, API and Infrastructure layers.
  • Create a Julia module.
  • Create the documentation for your module.
  • Let Julia communicate over a network.

Module A: Creating the model

Defining the Domain layer

We have chosen a simple model. Publishers send messages to subscribers. You can think an IoT application that consists of Raspberry Pi Zeros as publishers that send room temperatures to a central system, the subscriber. In Julia, you define objects as structs, e.g.

struct Subscriber
    name
    email
end # defined Subscriber object

A structs are immutable object. It means that you can't change its properties afterward. You can only read the values e.g

# Create a subscriber
daisy = Subscriber("Daisy", "daisy@duckcity.com")

# Print the name of the subscriber to the console
println( daisy.name )

# Reassign to change the object properties.
daisy = Subscriber("Daisy Duck", "daisy@duckcity.com")

Delaring enumrator data types

We could have subscribers who can do different jobs, e.g. calculate the average temperature of a data set, plotting it or even determine trends. Because we want clearly define the possible values we use enumerator data types, e.g.

@enum SubscriberType begin
    MEAN_CALCULATOR
    STD_CALCULATOR
    PLOTTER
end # defined enumeration for Subscriber type

We incorporate it into our Subscriber struct as follows, where we now also declare the data types of our properties. Types are declared by appending it with :: followed with the name of the type, e.g SubscriberType 

struct Subscriber
    name::String
    email::String
    subscribertype::SubscriberType
end # defined Subscriber object

Defining the functions for the API layer

We make use of an API layer to make it easier for the users to work with our model. Formally the API function have to be defined in the Domain layer and implemented in the API layer. WE define the next functions.

function createSubscriber end
function createPublisher end
function subscribe end
function unsubscribe end
function createMessage end
function sendMessage end

A future function could be:

function saveMessage end

We implement the function in the API layer.

Defining the functions for the Infrastructure layer

The infrastructure layer is our connection to the outer world. To be able to connect to different interface type we make use of the adapters. For example to save the message we can define adapters for in-memory storage or in a database.

function storeMsgInMemory end
function storeMsgInPostgreSQL end

We implement the two functions in de Infrastructure layer.

Implementing the Domain in domain.jl

We refer you to github for a complete overview. Here we show the implenetation of Subscriber including the help documentation.

"""
    struct Subscriber

# Arguments
- name::String
- email::String
- subscribertype::SubscriberType

Instanstiation: see: : [`createSubscriber()`](@ref)
Reference: RbO.Subscriber

# Example
julia> daisy = Subscriber("Daisy")
"""
struct Subscriber
    name::String
    email::String
    subscribertype::SubscriberType
    #constructors
    Subscriber( name::String ) = new( name, "", MEAN_CALCULATOR )
    Subscriber( name::String, email::String ) = new( name, email, MEAN_CALCULATOR )
    Subscriber( name::String, email::String, subscribertype::SubscriberType ) = new( name, email, subscribertype )
end # defined Scubscriber object

Implementing the API in api.jl

To hide details for the user we offer an API. There are three ways the user can instantiate a Subscriber object. Therefor we implement the three options in api.jl.

createSubscriber( name::String ) ::Subscriber =
    Subscriber( name )

createSubscriber( name::String, email::String ) ::Subscriber =
    Subscriber( name, email )

createSubscriber( name::String, email::String, subscribertype::SubscriberType )::Subscriber =
    Subscriber( name, email, subscribertype )
